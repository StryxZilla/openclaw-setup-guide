Get up and running with OpenClaw and start contributing to Arlo's projects

1. Prerequisites

Node.js (Required)

OpenClaw requires Node.js 18 or later.

# Check your Node version
node --version

# Should output v18.x.x or higher

If you need to install or update Node.js: nodejs.org

Git (Required)

You'll need Git to clone repos and contribute.

# Check if Git is installed
git --version

Download Git if needed: git-scm.com

GitHub Account + SSH Keys (Required for Contributing)

To contribute to Arlo's repos, you'll need:

  1. A GitHub account (sign up at github.com)
  2. SSH keys configured for Git authentication

Setting up SSH keys:

# Generate a new SSH key
ssh-keygen -t ed25519 -C "your_email@example.com"

# Start the SSH agent
eval "$(ssh-agent -s)"

# Add your key
ssh-add ~/.ssh/id_ed25519

# Copy your public key
cat ~/.ssh/id_ed25519.pub

Then add the public key to GitHub: Settings â†’ SSH and GPG keys â†’ New SSH key

Full guide: GitHub SSH Setup

Windows Users: OpenClaw works great on Windows! No need for WSL unless you prefer it. Use PowerShell or Git Bash for the commands above.

2. Install OpenClaw

Via npm (Recommended)

# Install globally
npm install -g openclaw

# Verify installation
openclaw --version

From Source (Alternative)

# Clone the repo
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Install dependencies
npm install

# Link globally
npm link

3. First Run & Configuration

Start the Gateway

openclaw gateway start

This will:

  • Create ~/.openclaw/ directory
  • Generate a default config file
  • Start the OpenClaw gateway service

Configure Your API Keys

Edit ~/.openclaw/openclaw.json to add your model provider API keys:

{
  "models": {
    "provider": "anthropic",  // or "openrouter", "openai", etc.
    "apiKey": "your-api-key-here"
  }
}
Tip: For testing/low-cost use, OpenRouter offers free models (Llama 3.3 70B). Anthropic Claude is the default for production work.

Workspace Setup

Your workspace lives in ~/.openclaw/workspace. Key files:

4. Basic Commands

# Check gateway status
openclaw status

# Start/stop/restart the gateway
openclaw gateway start
openclaw gateway stop
openclaw gateway restart

# Start a chat session
openclaw chat

# View help
openclaw help
API Costs: Be mindful of token usage. Use cheaper models for simple tasks. Check /status in chat to see usage stats.

5. Understanding Subagents

One of OpenClaw's killer features: subagents let you spawn background AI workers to handle tasks while you keep the conversation flowing.

When to use subagents:

  • Any task that takes >30 seconds of focused work
  • Building, coding, or research tasks
  • Anything where you want async progress updates
# Example: spawn a subagent to build something
sessions_spawn(
  task="Build a simple React app with...",
  label="react-app-build"
)

The subagent works in the background and pings you when done. Meanwhile, you can keep chatting or spawn more subagents.

Branch Workflow

Most repos use master or main as the primary branch. For contributions:

  1. Create a feature branch: git checkout -b feature/your-feature-name
  2. Make your changes and commit: git add . && git commit -m "Description"
  3. Push: git push origin feature/your-feature-name
  4. Open a Pull Request on GitHub
Important: These are Arlo's repos, so coordinate with Andy/Arlo before making major changes. PRs are reviewed before merging.

7. Tips & Gotchas

Signal Integration (Optional)

If you want OpenClaw to receive/send Signal messages, you'll need signal-cli. It's a bit involved â€” check the signal-cli docs if you're interested.

Managing API Costs

  • Use OpenRouter free models (Llama 3.3 70B) for simple tasks
  • Reserve Claude Opus/Sonnet for complex work
  • Check /status regularly to monitor token usage
  • Set budget alerts if your provider supports them

Common Troubleshooting

Gateway won't start:

  • Check if port 18789 is already in use
  • Try openclaw gateway stop && openclaw gateway start

Agent seems to have forgotten context:

  • Check workspace files (SOUL.md, USER.md, MEMORY.md) exist
  • Restart gateway to reload workspace

Commands not found:

  • Make sure npm global bin is in your PATH
  • On Windows, restart your terminal after install
