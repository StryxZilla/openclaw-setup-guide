OpenClaw Setup Guide

Get your AI agent running in 3 commands. Everything else is optional.

Quick Start

1 npm install -g openclaw
2 openclaw gateway start
3 openclaw chat

The Memory System

"Your agent wakes up fresh every session. These files are how it remembers who you are, what you're working on, and how to help you."

OpenClaw agents are stateless by default. Everything resets between sessions. The memory system solves this by giving your agent files to read at startup.

These files live in ~/.openclaw/workspace/ and are read automatically every session.

AGENTS.md â€” Workspace Notes

Instructions for the agent on how to use the workspace and memory system.

AGENTS.md 
# Workspace Notes

## Every Session
- Read SOUL.md, USER.md, MEMORY.md first
- Check memory/ folder for recent daily notes

## Memory Habits
- Write to memory/YYYY-MM-DD.md during conversations
- Update MEMORY.md with important stuff weekly

Create these files in ~/.openclaw/workspace/ before your first session. The agent will read them automatically.

Basic Commands

Gateway Management 
# Check status
openclaw status

# Start the gateway
openclaw gateway start

# Stop the gateway
openclaw gateway stop

# Restart the gateway
openclaw gateway restart
Chat & Sessions 
# Start a chat session
openclaw chat

# Use a specific model
openclaw chat --model anthropic/claude-opus-4

# View help
openclaw help
Configuration

Edit ~/.openclaw/openclaw.json to configure API keys, models, and settings.

openclaw.json 
{
  "models": {
    "provider": "anthropic",
    "apiKey": "your-api-key-here"
  }
}
